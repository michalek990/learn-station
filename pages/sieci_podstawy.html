Adresy IP
IPv4, IPv6, co oznaczają liczby
IP = adres domku w internecie. IPv4 to 32 bity zapisane jako 4 liczby 0–255 oddzielone kropkami. Każda liczba to jeden oktet (8 bitów). Dlatego max wartość to 255 (= 11111111 w binarnym).

Adresy prywatne nie są routowane przez internet — istnieją tylko lokalnie. NAT (router) tłumaczy je na jeden publiczny IP.
🌐 IPv6 — następca IPv4
IPv4: 32 bity → ~4.3 miliarda adresów (za mało!) IPv6: 128 bitów → 340 undecylionów adresów Zapis IPv6: 2001:0db8:85a3:0000:0000:8a2e:0370:7334 Skracanie (grupy zer można skrócić): 2001:db8::1 (:: = skrócone zera) ::1 (localhost w IPv6) fe80::1 (link-local, jak 169.254.x.x) Serwer na 0.0.0.0 = tylko IPv4 Serwer na :: = IPv4 + IPv6 (dual-stack)
🔄 NAT — jeden IP dla całego domu

P2P (gry, WebRTC) nienawidzi NAT — stąd STUN/TURN.

05
DNS — Domain Name System
jak nazwa domenowa zamienia się w adres IP
🔍 Jak działa zapytanie DNS krok po kroku
1
Lokalny cache
Przeglądarka i system sprawdzają czy już zna odpowiedź (TTL nie wygasł).
2
Resolver rekurencyjny
Pytasz serwer DNS swojego dostawcy internetu (lub 8.8.8.8 / 1.1.1.1).
Twój PC → 8.8.8.8: "Jaki IP ma google.com?"
3
Root server (.)
13 zestawów serwerów na świecie. Resolver pyta: "kto zna .com?"
Root → "serwery .com to: a.gtld-servers.net"
4
Serwer TLD (.com)
Serwer dla domeny najwyższego poziomu. Wie kto zarządza google.com.
TLD → "authoritative dla google.com to: ns1.google.com"
5
Authoritative Name Server
Serwer który naprawdę zna odpowiedź dla tej domeny.
ns1.google.com → "google.com = 142.250.185.78"
✓
Odpowiedź + TTL
Resolver zwraca IP i cache'uje wynik na czas TTL. Następnym razem od razu z cache'u.

Porty
drzwi do konkretnej usługi na komputerze
Port to 16-bitowa liczba (0–65535). Identyfikuje konkretną usługę/aplikację. Połączenie TCP identyfikuje czwórka: (IP źródłowy, port źródłowy, IP docelowy, port docelowy). Dlatego jeden serwer może obsługiwać tysiące jednoczesnych połączeń na tym samym porcie.

07
Gniazda (Sockets)
endpoint komunikacji sieciowej w kodzie
TCP socket — serwer i klient
// SERWER (Java)
ServerSocket server = new ServerSocket(8080);
while (true) {
  Socket client = server.accept();  // blokuje
  // każdy klient w nowym wątku
  InputStream  in  = client.getInputStream();
  OutputStream out = client.getOutputStream();
  // czytaj/pisz dane...
  client.close();
}

// KLIENT (Java)
Socket sock = new Socket("192.168.1.100", 8080);
OutputStream out = sock.getOutputStream();
out.write("Hello!".getBytes());
sock.close();
⚠️ TCP socket = strumień, nie wiadomości
TCP nie ma granic wiadomości! Możesz wysłać 1000 bajtów, a recv() zwróci może 300 + 700. Musisz sam zarządzać granicami — albo prefiks długości (int len + bajty), albo delimiter (\n), albo protokół wyższego rzędu (HTTP).
