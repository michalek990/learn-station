01 · spring security 6
Security — filtrowanie i autoryzacja
Spring Security 6 porzucił WebSecurityConfigurerAdapter na rzecz deklaratywnego SecurityFilterChain. Całość opiera się na łańcuchu filtrów HTTP — każde żądanie przechodzi przez filtry nim dotrze do kontrolera.
SecurityConfig.java
Spring Security 6 
@Configuration
@EnableWebSecurity
@EnableMethodSecurity       // @PreAuthorize etc.
public class SecurityConfig {

    @Bean
    public SecurityFilterChain filterChain(HttpSecurity http)
            throws Exception {
        return http
            .csrf(AbstractHttpConfigurer::disable)
            .sessionManagement(s -> s
                .sessionCreationPolicy(SessionCreationPolicy.STATELESS))
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/api/public/**").permitAll()
                .requestMatchers("/api/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated())
            .addFilterBefore(jwtFilter,
                UsernamePasswordAuthenticationFilter.class)
            .build();
    }

    @Bean
    public PasswordEncoder passwordEncoder() {
        return new BCryptPasswordEncoder(12);
    }
}
JwtFilter.java
OncePerRequestFilter 
@Component
public class JwtFilter extends OncePerRequestFilter {

    @Override
    protected void doFilterInternal(
            HttpServletRequest req,
            HttpServletResponse res,
            FilterChain chain) throws IOException, ServletException {

        String header = req.getHeader("Authorization");
        if (header != null && header.startsWith("Bearer ")) {
            String token = header.substring(7);
            if (jwtService.isValid(token)) {
                UsernamePasswordAuthenticationToken auth =
                    jwtService.getAuthentication(token);
                SecurityContextHolder.getContext()
                    .setAuthentication(auth);
            }
        }
        chain.doFilter(req, res);
    }
}

02 · spring webflux / project reactor
Reactive Programming
WebFlux to reaktywny stos Springa oparty na Project Reactor. Zamiast blokować wątki, używa nieblokujących strumieni danych — Mono<T> (0 lub 1 element) i Flux<T> (0..N elementów).
💡 Kiedy używać WebFlux? Używaj gdy masz dużo I/O (HTTP calls, bazy danych) i chcesz obsłużyć wielu użytkowników małą liczbą wątków. Nie używaj gdy aplikacja jest CPU-bound lub gdy cały stack nie jest reaktywny — mieszanie blokujące/reaktywne niszczy korzyści.
Mono & Flux — podstawy 
// Mono — pojedyncza wartość lub błąd
Mono<User> user = userRepo.findById(id);   // async

Mono<String> hello = Mono.just("Hello")
    .map(String::toUpperCase)
    .filter(s -> s.length() > 3)
    .defaultIfEmpty("EMPTY");

// Flux — strumień wielu wartości
Flux<Product> products = productRepo.findAll()
    .filter(Product::isActive)
    .take(10)
    .sort(Comparator.comparing(Product::getPrice));

// Łączenie strumieni
Mono<Order> order = userRepo.findById(userId)
    .zipWith(productRepo.findById(productId))
    .map(tuple -> new Order(tuple.getT1(), tuple.getT2()))
    .flatMap(orderRepo::save);
ReactiveController.java 
@RestController
@RequestMapping("/api/reactive")
public class ProductController {

    @GetMapping("/{id}")
    public Mono<Product> getProduct(@PathVariable Long id) {
        return productService.findById(id)
            .switchIfEmpty(Mono.error(
                new ResponseStatusException(HttpStatus.NOT_FOUND)));
    }

    @GetMapping(produces = MediaType.TEXT_EVENT_STREAM_VALUE)
    public Flux<Product> streamProducts() {
        return productRepo.findAll()    // Reactive repo
            .delayElements(Duration.ofMillis(100)); // SSE
    }

    @GetMapping("/webclient")
    public Mono<String> callExternal() {
        return WebClient.create("https://api.example.com")
            .get().uri("/data")
            .retrieve()
            .bodyToMono(String.class);
    }
}

03 · aspect oriented programming
AOP — aspekty przekrojowe
AOP pozwala wydzielić logikę przekrojową (logowanie, audyt, metryki, transakcje) z klas biznesowych. Spring AOP działa przez proxy — jest prostszy niż AspectJ compile-time, ale ograniczony do Spring beans.
LoggingAspect.java 
@Aspect
@Component
public class LoggingAspect {

    // Pointcut — wszystkie metody w service layer
    @Pointcut("execution(* com.app.service.*.*(..))")
    public void serviceLayer() {}

    // Before — przed wywołaniem metody
    @Before("serviceLayer()")
    public void logBefore(JoinPoint jp) {
        log.info("→ {} args: {}",
            jp.getSignature().getName(),
            jp.getArgs());
    }

    // Around — pełna kontrola wywołania
    @Around("@annotation(Timed)")
    public Object measureTime(ProceedingJoinPoint pjp)
            throws Throwable {
        long start = System.currentTimeMillis();
        Object result = pjp.proceed();  // wywołaj metodę
        long elapsed = System.currentTimeMillis() - start;
        log.info("⏱ {} took {}ms",
            pjp.getSignature().getName(), elapsed);
        return result;
    }
}
Rodzaje Advice
@Before
Przed metodą. Nie może zmienić zwracanej wartości
@After
Po metodzie — zawsze (sukces i wyjątek)
@AfterReturning
Po sukcesie. Dostęp do wartości zwracanej przez returning
@AfterThrowing
Po wyjątku. Dostęp do wyjątku przez throwing
@Around
Pełna kontrola — możesz nie wywołać metody, zmienić argumenty i wynik

04 · spring cache abstraction
Caching — Cache Abstraction
Spring Cache Abstraction pozwala dodać cachowanie przez adnotacje, niezależnie od implementacji (Caffeine, Redis, EhCache). Zmiana backendu nie wymaga zmian w kodzie biznesowym.
CacheableService.java 
@Service
public class ProductService {

    // Cachuje wynik, klucz = id
    @Cacheable(value = "products", key = "#id")
    public Product findById(Long id) {
        return repo.findById(id).orElseThrow();
    }

    // Aktualizuje cache po zapisie
    @CachePut(value = "products", key = "#result.id")
    public Product save(Product p) {
        return repo.save(p);
    }

    // Usuwa z cache po usunięciu
    @CacheEvict(value = "products", key = "#id")
    public void deleteById(Long id) {
        repo.deleteById(id);
    }

    // Czyści cały cache
    @CacheEvict(value = "products", allEntries = true)
    public void clearAll() {}
}

⚠️ Pułapki cache Cache miss storm: wiele requestów naraz przy zimnym cache. Stale data: pamiętaj o TTL i eviction. Self-invocation: @Cacheable nie działa przy wywołaniu wewnątrz tej samej klasy (brak proxy).
Redis Cache — zaawansowana konfiguracja
@Configuration
public class RedisCacheConfig {

    @Bean
    public RedisCacheManager cacheManager(RedisConnectionFactory factory) {
        RedisCacheConfiguration defaultConfig = RedisCacheConfiguration.defaultCacheConfig()
            .entryTtl(Duration.ofMinutes(10))
            .serializeValuesWith(RedisSerializationContext.SerializationPair
                .fromSerializer(new GenericJackson2JsonRedisSerializer()));

        return RedisCacheManager.builder(factory)
            .cacheDefaults(defaultConfig)
            .withCacheConfiguration("products",
                defaultConfig.entryTtl(Duration.ofHours(1)))
            .withCacheConfiguration("sessions",
                defaultConfig.entryTtl(Duration.ofMinutes(30)))
            .build();
    }
}

05 · resilience4j
Resilience4j — odporność systemu
Resilience4j to lekka biblioteka odporności zastępująca Netflix Hystrix. Dostarcza Circuit Breaker, Rate Limiter, Retry, Bulkhead i TimeLimiter — wszystko przez adnotacje lub konfigurację.
Circuit Breaker + Retry 
@Service
public class PaymentService {

    @CircuitBreaker(name = "payment", fallbackMethod = "paymentFallback")
    @Retry(name = "payment")
    @TimeLimiter(name = "payment")
    public CompletableFuture<PaymentResult> pay(PaymentRequest req) {
        return CompletableFuture.supplyAsync(
            () -> externalPaymentApi.process(req));
    }

    // Fallback — wywołany gdy CB jest OPEN
    public CompletableFuture<PaymentResult> paymentFallback(
            PaymentRequest req, Exception ex) {
        log.warn("CB OPEN: {}", ex.getMessage());
        return CompletableFuture.completedFuture(
            PaymentResult.queued(req));  // kolejkuj async
    }
}
application.yml — Resilience4j 
resilience4j:
  circuitbreaker:
    instances:
      payment:
        slidingWindowSize: 10
        failureRateThreshold: 50      # % błędów → OPEN
        waitDurationInOpenState: 10s  # czas w OPEN
        permittedCallsInHalfOpenState: 3
        slowCallDurationThreshold: 2s
        slowCallRateThreshold: 80

  retry:
    instances:
      payment:
        maxAttempts: 3
        waitDuration: 500ms
        exponentialBackoffMultiplier: 2
        retryExceptions:
          - java.io.IOException

  ratelimiter:
    instances:
      payment:
        limitForPeriod: 100
        limitRefreshPeriod: 1s
        timeoutDuration: 100ms

06 · apache kafka / spring kafka
Kafka — messaging i event streaming
Spring Kafka abstrahuje producenta i konsumenta Apache Kafka. Idealny do event-driven architecture, CQRS i komunikacji między mikroserwisami bez bezpośredniego sprzężenia (coupling).
Producer — wysyłanie wiadomości 
@Service
public class OrderEventProducer {

    private final KafkaTemplate<String, OrderEvent> kafkaTemplate;

    public void publishOrderCreated(Order order) {
        OrderEvent event = OrderEvent.builder()
            .orderId(order.getId())
            .userId(order.getUserId())
            .total(order.getTotal())
            .timestamp(Instant.now())
            .build();

        kafkaTemplate.send("orders", order.getId().toString(), event)
            .whenComplete((result, ex) -> {
                if (ex != null) {
                    log.error("Failed to send: {}", ex.getMessage());
                } else {
                    log.info("Sent to partition {}",
                        result.getRecordMetadata().partition());
                }
            });
    }
}
Consumer — odbieranie wiadomości 
@Component
public class OrderEventConsumer {

    @KafkaListener(
        topics = "orders",
        groupId = "inventory-service",
        containerFactory = "kafkaListenerContainerFactory")
    public void onOrderCreated(
            @Payload OrderEvent event,
            @Header(KafkaHeaders.RECEIVED_PARTITION) int partition,
            @Header(KafkaHeaders.OFFSET) long offset) {

        log.info("Received: {} from partition {} @ {}",
            event.getOrderId(), partition, offset);

        inventoryService.reserveStock(event);
    }

    // Dead Letter Topic — obsługa błędów
    @KafkaListener(topics = "orders.DLT", groupId = "dlt-handler")
    public void handleDlt(ConsumerRecord<String, OrderEvent> record) {
        log.error("DLT: {}", record.value());
        alertService.notifyTeam(record.value());
    }
}

Testy — od unit do integracyjnych
Spring Boot oferuje pełne wsparcie testów: od lekkich unit testów bez kontekstu, przez slice testy (tylko warstwy web/data), po pełne testy integracyjne z @SpringBootTest i Testcontainers.
@WebMvcTest — test kontrolera 
@WebMvcTest(UserController.class)
class UserControllerTest {

    @Autowired
    private MockMvc mockMvc;

    @MockBean
    private UserService userService;

    @Test
    void shouldReturnUser() throws Exception {
        var user = new User(1L, "Jan Kowalski");
        given(userService.findById(1L)).willReturn(user);

        mockMvc.perform(get("/api/users/1")
            .contentType(MediaType.APPLICATION_JSON))
            .andExpect(status().isOk())
            .andExpect(jsonPath("$.name").value("Jan Kowalski"))
            .andExpect(jsonPath("$.id").value(1));
    }

    @Test
    void shouldReturn404WhenNotFound() throws Exception {
        given(userService.findById(999L))
            .willThrow(new UserNotFoundException(999L));

        mockMvc.perform(get("/api/users/999"))
            .andExpect(status().isNotFound());
    }
}

Własne metryki — Micrometer 
@Service
public class OrderService {

    private final Counter orderCounter;
    private final Timer processingTimer;

    public OrderService(MeterRegistry registry) {
        this.orderCounter = Counter.builder("orders.created")
            .tag("service", "orders")
            .description("Total orders created")
            .register(registry);

        this.processingTimer = Timer.builder("orders.processing.time")
            .publishPercentiles(0.5, 0.95, 0.99)
            .register(registry);
    }

    public Order createOrder(OrderRequest req) {
        return processingTimer.record(() -> {
            Order order = processOrder(req);
            orderCounter.increment();
            return order;
        });
    }
}

09 · oauth2 / jwt / openid connect
OAuth2 — Resource Server & JWT
Spring Security integruje OAuth2 Resource Server — weryfikuje JWT tokeny z zewnętrznego Identity Provider (Keycloak, Auth0, Okta). Aplikacja nigdy nie zarządza hasłami bezpośrednio.
Resource Server Config 
@Configuration
@EnableWebSecurity
public class ResourceServerConfig {

    @Bean
    public SecurityFilterChain securityChain(HttpSecurity http) throws Exception {
        return http
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/public/**").permitAll()
                .anyRequest().authenticated())
            .oauth2ResourceServer(oauth2 -> oauth2
                .jwt(jwt -> jwt.jwtAuthenticationConverter(
                    jwtAuthenticationConverter())))
            .build();
    }

    private JwtAuthenticationConverter jwtAuthenticationConverter() {
        var converter = new JwtGrantedAuthoritiesConverter();
        converter.setAuthoritiesClaimName("roles");  // Keycloak claim
        converter.setAuthorityPrefix("ROLE_");

        var authConverter = new JwtAuthenticationConverter();
        authConverter.setJwtGrantedAuthoritiesConverter(converter);
        return authConverter;
    }
}

Dostęp do claims JWT 
@GetMapping("/me")
public UserInfo me(@AuthenticationPrincipal Jwt jwt) {
    String sub = jwt.getSubject();
    String email = jwt.getClaimAsString("email");
    List<String> roles = jwt.getClaimAsStringList("roles");
    return new UserInfo(sub, email, roles);
}

Wdrożenia — od JAR do K8s
Spring Boot doskonale integruje się z konteneryzacją. Buildpacks tworzą zoptymalizowane obrazy bez pisania Dockerfile. Na K8s używaj Actuator probes, ConfigMaps i Secrets przez Spring Cloud Kubernetes.

FROM eclipse-temurin:21-jdk-alpine AS build
WORKDIR /app
COPY mvnw pom.xml ./
COPY .mvn .mvn
RUN ./mvnw dependency:go-offline # cache dependencies

RUN ./mvnw package -DskipTests

# Etap 2: extract layers (Spring Boot layertools)
FROM eclipse-temurin:21-jdk-alpine AS layers

# Etap 3: runtime — minimalne zależności
FROM eclipse-temurin:21-jre-alpine
WORKDIR /app

COPY --from=layers /app/spring-boot-loader/ ./

Optymalizacje JVM

JAVA_OPTS w kontenerze:

-XX:MaxRAMPercentage=75
Limit heap jako % RAM kontenera. Bezpieczniejsze niż -Xmx

-XX:+UseContainerSupport
JVM odczytuje limity cgroups (domyślnie od Java 10+)

-XX:+UseZGC
Low-latency GC — pauzy <1ms, dobry dla usług real-time

-Djava.security.egd=...
