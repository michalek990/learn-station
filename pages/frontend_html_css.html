HTML — HyperText Markup Language
Struktura dokumentu i podstawowe znaczniki
HTML to szkielet każdej strony. Przeglądarka parsuje HTML i buduje DOM — drzewo obiektów. Każdy element to znacznik otwierający + treść + znacznik zamykający (lub samozamykający). Atrybuty dodają dodatkowe informacje.
Kompletna struktura dokumentu HTML5
<!DOCTYPE html>  <!-- deklaracja HTML5 — zawsze pierwsza! -->
<html lang="pl">  <!-- lang ważny dla SEO i czytników -->
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <meta name="description" content="Opis strony — ważny dla SEO, 150-160 znaków">
  <meta name="robots" content="index, follow">
  <!-- Open Graph — wygląd przy udostępnianiu w social -->
  <meta property="og:title" content="Tytuł">
  <meta property="og:description" content="Opis">
  <meta property="og:image" content="thumbnail.jpg">
  <meta property="og:type" content="website">
  <title>Tytuł strony — widoczny w zakładce i Google</title>
  <!-- CSS —  w <head> żeby uniknąć FOUC -->
  <link rel="stylesheet" href="style.css">
  <link rel="icon" href="favicon.ico">
  <link rel="preconnect" href="https://fonts.googleapis.com">
</head>
<body>
  <!-- cała widoczna treść tutaj -->

  <!-- JS na końcu body lub z defer — nie blokuje renderowania -->
  <script src="app.js" defer></script>
  <script src="analytics.js" async></script>
  <!-- defer: zachowuje kolejność, czeka na DOM
       async: ładuje równolegle, wykonuje natychmiast -->
</body>
</html>

🎭 Kontenery ogólne
<div>
Block container — bez semantyki, layout i stylowanie. Najczęściej używany.
<span>
Inline container — bez semantyki, wewnątrz tekstu. Do stylowania fragmentów.
<details>
Rozwijana sekcja. Z <summary> jako nagłówkiem. Accordion bez JS!
<dialog>
Natywny modal. .showModal() w JS. ::backdrop pseudo-element tła.
<template>
HTML nierendowany — klon przez JS do dynamicznych treści.
<canvas>
Obszar rysowania 2D/WebGL przez JavaScript API.
⚡ Atrybuty globalne HTML
id
Unikalny identyfikator. Cel linków anchor, hook JS.
class
Klasy CSS (wiele oddzielonych spacją). Selektor CSS/JS.

Własne dane: data-id="5" → JS: el.dataset.id
style
Inline CSS. Najwyższa specyficzność — unikaj!
hidden
Ukrywa element (jak display:none).
tabindex
0=fokus tabem, -1=fokus JS, 1+=kolejność (unikaj).

Edytowalny element w przeglądarce.
draggable
Drag and Drop API.

HTML5 Semantyka
Znaczniki semantyczne — struktura ze znaczeniem
Semantyczny HTML opisuje znaczenie treści. Ważny dla SEO (Google rozumie strukturę), dostępności (screen readery nawigują po landmarks) i czytelności kodu. Zamiast "div soup" — odpowiednie znaczniki.
Pełny układ semantyczny strony
<body>
  <header>  <!-- nagłówek strony lub sekcji -->
    <nav aria-label="Główna nawigacja">
      <ul><li><a href="/">Strona główna</a></li></ul>
    </nav>
  </header>

  <main>  <!-- JEDEN na stronie — główna treść -->

    <section>  <!-- tematyczna sekcja — zawsze z nagłówkiem -->
      <h2>Tytuł sekcji</h2>

      <article>  <!-- samodzielna treść (post, karta, komentarz) -->
        <header>
          <h3>Tytuł artykułu</h3>
          <time datetime="2024-06-15T10:00">15 czerwca 2024</time>
          <address>Autor: <a href="mailto:[email protected]">Jan Kowalski</a></address>
        </header>
        <p>Treść...</p>
        <figure>
          <img src="chart.png" alt="Wykres sprzedaży">
          <figcaption>Wykres 1. Wzrost sprzedaży 2024</figcaption>
        </figure>
        <footer>
          <p>Tagi: <a href="/css" rel="tag">CSS</a></p>
        </footer>
      </article>

    </section>

    <aside aria-label="Powiązane treści">  <!-- sidebar -->
      <h2>Powiązane artykuły</h2>
    </aside>

  </main>

  <footer>  <!-- stopka strony -->
    <p><small>&copy; 2024 Firma. Wszelkie prawa zastrzeżone.</small></p>
  </footer>
</body>
🏷️ Kompletny katalog znaczników semantycznych
ZnacznikZastosowanie
<header>Nagłówek strony lub sekcji. Logo, nav, h1.
<nav>Blok linków nawigacyjnych. Menu, breadcrumbs, paginacja.
<main>Główna, unikalna treść. Jeden per strona.
<article>Samodzielna treść (post, tweet, karta produktu, komentarz).
<section>Tematyczna grupa treści. Wymaga nagłówka.
<aside>Poboczna treść (sidebar, reklama, powiązane linki).
<footer>Stopka strony lub sekcji. Prawa autorskie, linki.
<figure>Ilustracja, wykres, kod — z <figcaption>.
<time>Data/czas. datetime atrybut dla maszyn.
<address>Dane kontaktowe autora lub organizacji.
<details>Rozwijana sekcja. <summary> jako widoczny nagłówek.
<dialog>Natywny modal/dialog. API: .showModal().
<search>Obszar wyszukiwania (HTML 5.3, nowy!).
<hgroup>Grupuje nagłówek z podtytułem.
💡 Div vs Section vs Article — kiedy co? div: brak semantyki, czysty layout/hook JS.
section: tematyczna część strony — zawsze z h2/h3.
article: samodzielna treść którą można skopiować do innego kontekstu.
Reguła: "Czy ten element ma znaczenie?" → tak: semantyczny. Nie: div.

Formularze HTML5
Formularze — wszystkie typy inputów i walidacja
HTML5 dodał wiele nowych typów inputów (email, date, range, color...) i natywną walidację. Prawidłowe formularze są kluczowe dla dostępności — każdy input musi mieć label.
Kompletny formularz — wszystkie typy inputów
<form action="/submit" method="post"
      enctype="multipart/form-data">  <!-- wymagane dla plików -->

  <!-- label FOR powiązany z input ID! -->
  <label for="email">Email:</label>
  <input type="email" id="email" name="email"
         required placeholder="[email protected]"
         autocomplete="email">

  <!-- Wszystkie typy -->
  <input type="text"     minlength="3" maxlength="50" pattern="[A-Za-z]+">
  <input type="password"  autocomplete="new-password">
  <input type="number"   min="0" max="100" step="5">
  <input type="date"     min="2020-01-01" max="2024-12-31">
  <input type="time"     step="300">  <!-- co 5 minut -->
  <input type="datetime-local">
  <input type="week">  <input type="month">
  <input type="range"    min="0" max="100" value="50">
  <input type="color"    value="#e44d26">
  <input type="file"     accept="image/*,.pdf" multiple>
  <input type="checkbox" name="agree" value="yes" checked>
  <input type="radio"    name="group" value="a">
  <input type="search"   list="suggestions">
  <input type="tel"      pattern="[0-9]{9}">
  <input type="url">
  <input type="hidden"   value="csrf_token_123">

  <textarea rows="5" resize="vertical"></textarea>

  <select name="country" multiple>
    <optgroup label="Europa">
      <option value="pl" selected>Polska</option>
    </optgroup>
  </select>

  <!-- Datalist = natywne autocomplete -->
  <datalist id="suggestions">
    <option value="Warszawa">  <option value="Kraków">
  </datalist>

  <fieldset>
    <legend>Dane osobowe</legend>
    <!-- grupuje powiązane pola -->
  </fieldset>

  <button type="submit">Wyślij</button>
  <button type="reset">Resetuj</button>
  <button type="button" onclick="doStuff()">Akcja JS</button>
</form>
Walidacja — atrybuty i CSS
<!-- ─── Atrybuty walidacji ─────────────────────────────── -->
required              <!-- pole wymagane -->
minlength="3"         <!-- min znaków (text, password) -->
maxlength="100"       <!-- max znaków -->
min="0" max="100"      <!-- zakres liczbowy -->
pattern="[A-Za-z]{3,}" <!-- regex (bez // i flag) -->
step="0.01"            <!-- krok liczby/daty -->
accept="image/png,image/jpeg" <!-- typy pliku -->
disabled               <!-- wyłączone — nie wysyłane! -->
readonly               <!-- tylko do odczytu — wysyłane -->
novalidate             <!-- na form — wyłącza walidację -->

input:valid            { border-color: green; }
input:invalid          { border-color: red; }
input:required         { border-left: 3px solid orange; }
input:optional         { opacity: .8; }
input:focus            { outline: 2px solid blue; }
input:disabled         { opacity: .4; cursor: not-allowed; }
input:read-only        { background: #f5f5f5; }
input:placeholder-shown{ background: lightyellow; }
input:user-invalid     { border-color: red; } /* po interakcji */

CSS — Cascading Style Sheets
Kaskada, specyficzność, dziedziczenie, jednostki
CSS decyduje o wyglądzie. Kaskada, specyficzność i dziedziczenie to trzy fundamenty. Zrozumienie ich usuwa "nie wiem dlaczego moje style nie działają". Jednostki relatywne (rem, em, vw) to podstawa responsive design.
Specyficzność i kaskada
/* Specyficzność = (A, B, C):
   A = #id selectors
   B = .class, [attr], :pseudo-class
   C = element, ::pseudo-element
   Większa = wygrywa. Przy równej = OSTATNIA wygrywa */

*             /* (0,0,0) — universal */
p             /* (0,0,1) — element */
.card         /* (0,1,0) — class */
#header       /* (1,0,0) — ID */
p.card        /* (0,1,1) */
#header .nav a/* (1,1,1) */
/* style="" → inline — bije selektory */
color: red !important; /* nuclear — unikaj! */

/* Kolejność kaskady (co wygrywa):
   1. !important browser
   2. !important user
   3. !important author
   4. Inline style
   5. ID selector
   6. Class/attr/pseudo-class
   7. Element/pseudo-element
   8. Browser defaults */

/* Wartości kaskady */
color: inherit;  /* przejmij od rodzica */
color: initial;  /* domyślna przeglądarki */
color: unset;    /* inherit jeśli dziedziczone, else initial */
color: revert;   /* cofnij do stylu user-agent */
all: revert;     /* reset WSZYSTKICH właściwości */

/* Co się DZIEDZICZY:
   color, font-*, line-height, text-*, cursor, visibility
   Co NIE:
   background, border, margin, padding, width, height */
Kolory i jednostki — wszystkie typy
/* ─── KOLORY ──────────────────────────────────────────── */

Box Model & Display
Model pudełkowy — margin, border, padding, content
Każdy element HTML to prostokąt. Box Model definiuje jak zajmuje miejsce. box-sizing: border-box to najważniejsza właściwość CSS — zawsze jej używaj. Display określa jak element zachowuje się w flow.
MARGIN — przestrzeń zewnętrzna (transparentna, background nie widoczny)
BORDER — obramowanie (tło widoczne)
PADDING — wewnętrzna przestrzeń (tło rodzica widoczne)
CONTENT — tekst, img, dzieci
Box Model właściwości
/* NAJWAŻNIEJSZE — zawsze stosuj! */
*, *::before, *::after { box-sizing: border-box; }
/* content-box (domyślnie): width = tylko content
   border-box: width = content + padding + border */

/* Margin — shorthand (clock: góra, prawo, dół, lewo) */
margin: 10px;              /* wszystkie strony */
margin: 10px 20px;         /* góra/dół, lewo/prawo */
margin: 5px 10px 15px;     /* góra, lewo/prawo, dół */
margin: 5px 10px 15px 20px;/* góra, prawo, dół, lewo */
margin: auto;             /* centrowanie block elementów! */
margin-top: -10px;        /* ujemny margin = nakładanie */
/* Margin Collapse! Dwa sąsiednie bloki: bierze większy */

/* Border */
border: 2px solid #333;
border-style: solid|dashed|dotted|double|none;
border-radius: 8px;
border-radius: 50%;        /* koło */
border-radius: 10px 20px 30px 5px; /* per róg */
outline: 2px solid blue;  /* poza border, nie zajmuje miejsca */
outline-offset: 4px;
box-shadow: 0 4px 12px rgba(0,0,0,.2);
box-shadow: inset 0 2px 4px rgba(0,0,0,.1); /* wewnętrzny */
Display — zachowanie elementów
display: block;
/* Zajmuje pełną szerokość rodzica
   Nowa linia przed i po. Width/height działa.
   Elementy: div, p, h1-h6, ul, form, section */

display: inline;
/* Tylko tyle miejsca ile treść. Nie łamie.
   BRAK width/height, pionowy margin/padding ignorowany
   Elementy: span, a, strong, em */

display: inline-block;
/* Inline (nie łamie) + block (width/height działa)
   Przydatne: przyciski, badge, ikonki */

display: none;
/* Ukryty — NIE zajmuje miejsca
   vs visibility:hidden — ukryty ale zajmuje miejsce
   vs opacity:0 — niewidoczny, zajmuje, reaguje na eventy */

display: flex;   /* Flexbox container */
display: grid;   /* Grid container */
display: contents; /* "Znikający" wrapper */
display: list-item; /* jak li — ma bullet */

/* ─── Overflow ────────────────────────────────────────── */
overflow: visible;  /* domyślnie — wychodzi poza */
overflow: hidden;   /* obcina — tworzy BFC! */
overflow: scroll;   /* zawsze scrollbar */
overflow: auto;     /* scrollbar gdy potrzeba ✓ */
overflow-clip-margin: 8px; /* margines przed clip */

/* Obcięcie jednej linii z "..." */
.truncate {
  white-space: nowrap;
  overflow: hidden;
  text-overflow: ellipsis;
}
/* Wieloliniowe "..." — line-clamp */
.clamp-3 {
  display: -webkit-box;
  -webkit-line-clamp: 3;
  -webkit-box-orient: vertical;
  overflow: hidden;
}

Selektory i Pseudo-elementy
Selektory, pseudo-klasy, pseudo-elementy
Selektory to sposób celowania w elementy. Od prostych przez kombinatory, atrybuty, pseudo-klasy strukturalne (:nth-child, :has) po pseudo-elementy tworzące wirtualne węzły DOM (::before, ::after).
Kompletne selektory CSS
/* ─── Podstawowe ─────────────────────────────────────── */
*              { } /* universal */
div            { } /* element */
.class         { } /* class */
#id            { } /* ID — unikaj w CSS! */
div, p, span   { } /* lista selektorów */

/* ─── Kombinatory ───────────────────────────────────── */
div p          { } /* descendant — p w dowolnym div */
div > p        { } /* direct child — tylko bezpośredni */
h2 + p         { } /* adjacent sibling — p tuż po h2 */
h2 ~ p         { } /* general sibling — wszystkie p po h2 */

/* ─── Atrybuty ──────────────────────────────────────── */
[href]            { } /* ma atrybut href */
[type="text"]     { } /* href = wartość */
[class~="card"]   { } /* słowo card w class */
[href^="https"]   { } /* zaczyna się od https */
[href$=".pdf"]    { } /* kończy się .pdf */
[href*="google"]  { } /* zawiera google */
[lang|="pl"]      { } /* pl lub pl-PL */
[type="text" i]  { } /* case-insensitive */

/* ─── Pseudo-klasy — stan ───────────────────────────── */
:hover            /* kursor nad elementem */
:focus            /* element w fokusie */
:focus-visible    /* fokus z klawiatury (nie klik) */
:focus-within     /* potomek ma fokus */
:active           /* kliknięty */
:visited          /* odwiedzony link */
:target           /* el który jest celem URL #anchor */
:checked          /* checkbox/radio zaznaczony */
:indeterminate    /* checkbox w stanie pośrednim */

/* ─── Pseudo-klasy — strukturalne ──────────────────── */
:first-child       /* pierwsze dziecko */
:last-child        /* ostatnie dziecko */
:nth-child(2)      /* drugie dziecko */
:nth-child(odd)    /* nieparzyste */
:nth-child(even)   /* parzyste */
:nth-child(3n+1)   /* 1,4,7,10... */
:nth-child(-n+3)   /* pierwsze 3 */
:first-of-type     /* pierwszy element tego typu */
:only-child        /* jedyne dziecko */
:empty             /* bez dzieci ani tekstu */
:not(.active)      /* NOT selector */
:is(h1,h2,h3)      /* który z listy — bierze max specyficzność */
:where(h1,h2)      /* jak :is ale specyficzność 0 */
:has(img)          /* rodzic który MA img ← game changer! */
:has(+ p)          /* ma bezpośrednio po sobie p */
Pseudo-elementy — wirtualne węzły DOM
/* Tworzą "wirtualne" elementy (nie w DOM)
   Zawsze :: (dwa dwukropki) w CSS3 */

.el::before {
  content: "▶ ";          /* tekst przed */
  content: "";             /* pusty — do kształtów */
  content: attr(data-label);/* wartość atrybutu */
  content: url("icon.svg"); /* obraz */
  display: block;
  width: 20px; height: 20px;
}
.el::after  { content: ""; } /* za elementem */

/* Inne pseudo-elementy: */
p::first-line   { font-weight: bold; }
p::first-letter {               /* dropcap */
  font-size: 3em; float: left;
  line-height: 1;
}
::selection     { background: yellow; color: black; }
::placeholder   { color: gray; opacity: .7; }
::marker        { color: red; font-size: 1.2em; }
::backdrop      { background: rgba(0,0,0,.7); }
::scrollbar          { width: 8px; }
::scrollbar-track    { background: #f1f1f1; }
::scrollbar-thumb    { background: #888; border-radius: 4px; }
::-webkit-scrollbar { } /* Chrome/Safari prefix */

/* ─── Tooltip bez JS ──────────────────────────────────── */
[data-tip] { position: relative; }
[data-tip]:hover::after {
  content: attr(data-tip);
  position: absolute;
  bottom: 100%; left: 50%;
  transform: translateX(-50%);
  background: #333; color: white;
  padding: 4px 8px; border-radius: 4px;
  white-space: nowrap; font-size: .85rem;

Flexbox
CSS Flexbox — jednowymiarowy layout
Flexbox to układ jednowymiarowy — wiersz lub kolumna. Idealny dla komponentów: nawigacja, karty, przyciski, centra. Dwa zestawy właściwości: kontener (flex-direction, justify-content, align-items) i elementy (flex, order, align-self).
Flexbox — kontener (parent)
.flex-container {
  display: flex;  /* lub inline-flex */

  /* Oś główna */
  flex-direction: row;            /* → domyślnie */
  flex-direction: row-reverse;   /* ← */
  flex-direction: column;        /* ↓ */
  flex-direction: column-reverse;/* ↑ */

  /* Zawijanie */
  flex-wrap: nowrap;    /* domyślnie — jedna linia */
  flex-wrap: wrap;      /* zawijaj do nowych linii */
  flex-wrap: wrap-reverse;
  flex-flow: row wrap; /* skrót direction + wrap */

  /* Wyrównanie na OSI GŁÓWNEJ */
  justify-content: flex-start;    /* domyślnie */
  justify-content: flex-end;
  justify-content: center;
  justify-content: space-between; /* równe odstępy między */
  justify-content: space-around;  /* odstępy wokół */
  justify-content: space-evenly;  /* identyczne odstępy */

  /* Wyrównanie na OSI POPRZECZNEJ */
  align-items: stretch;    /* domyślnie */
  align-items: flex-start;
  align-items: flex-end;
  align-items: center;     /* ← pionowe centrowanie! */
  align-items: baseline;   /* linia bazowa tekstu */

  /* Gdy flex-wrap — dla wielu linii */
  align-content: space-between;

  /* Odstępy — nowoczesne */
  gap: 1rem;
  gap: 1rem 2rem; /* row-gap col-gap */
  row-gap: 1rem;
  column-gap: 2rem;
}
Flexbox — elementy (children) + wzorce
.flex-item {
  /* flex = flex-grow flex-shrink flex-basis */
  flex: 1;           /* = 1 1 0 — rośnie równo */
  flex: auto;        /* = 1 1 auto */
  flex: none;        /* = 0 0 auto — nie zmienia */
  flex: 0 0 200px;  /* stały 200px */

  entries.forEach(e => {
