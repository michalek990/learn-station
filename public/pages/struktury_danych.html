Tablica (Array)
indeksowany, ciągły blok pamięci

Elementy leżą obok siebie w pamięci. Znasz indeks → masz element natychmiast. Jak szuflady ponumerowane od 0.

int[] arr = {7, 2, 9, 4, 1, 6, 3}

dostęp [i] → O(1) zapis [i] → O(1) szukanie → O(n) wstawianie → O(n)

Stos (Stack)
LIFO — Last In, First Out

Jak stos talerzy — kładziesz na wierzch (push) i zdejmujesz z wierzchu (pop). Ostatni wchodzi, pierwszy wychodzi.

push → O(1) pop → O(1) peek → O(1) szukanie → O(n)

🌳
Drzewo Binarne (BST)
lewy syn < rodzic < prawy syn

Każdy węzeł ma max 2 dzieci. W BST: lewe dziecko jest mniejsze, prawe większe. Szukanie dzieli problem na pół — O(log n).

szukaj 62: 50→prawo→75→lewo→62 ✓ (3 kroki!)
szukaj → O(log n) wstaw → O(log n) usuń → O(log n) najgorzej → O(n)

Hash Mapa (HashMap)
klucz → hash → indeks → wartość

Klucz przechodzi przez funkcję hash która daje indeks w tablicy. Dostęp do wartości po kluczu — prawie zawsze O(1).

🕸️
Graf (Graph)
węzły + krawędzie, skierowany/nieskierowany

Węzły (vertices) połączone krawędziami (edges). Modeluje sieci, drogi, zależności. Przeszukiwanie: BFS lub DFS.

BFS → O(V+E) DFS → O(V+E) sąsiedzi → O(1) najkrótsza → O(E log V)
⛰️
Kopiec (Heap)
max-heap: rodzic ≥ dzieci zawsze

Drzewo gdzie rodzic jest zawsze większy (max-heap) lub mniejszy (min-heap) od dzieci. Błyskawiczny dostęp do max/min.

MAX-HEAP — korzeń zawsze największy
peek max → O(1) wstaw → O(log n) usuń max → O(log n) szukaj → O(n)
📈
Big O — porównanie złożoności
dla n = 1000 elementów

Jak szybko rośnie liczba operacji gdy dane rosną. Im niższy słupek tym lepiej.

O(1)
1 op
O(log n)
~10 ops
O(n)
1K ops
O(n log n)
~10K ops
O(n²)
1M ops
O(2ⁿ)
∞ ops 💀
O(1) — dostęp do tablicy, HashMap O(log n) — binary search, BST O(n) — pętla, szukanie liniowe O(n log n) — merge sort, quick sort O(n²) — bubble sort, pętle zagnieżdżone O(2ⁿ) — naiwna rekurencja fib
