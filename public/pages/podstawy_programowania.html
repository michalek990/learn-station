01 / 09 ⚡ Absolutne podstawy

Komputer to tylko prąd i tranzystory

Żeby rozumieć programowanie, musisz zrozumieć jedną rzecz: komputer nie jest inteligentny. To maszyna która potrafi tylko jedno — sprawdzić czy przez jakiś drucik płynie prąd, czy nie płynie. To wszystko. Reszta to magia abstrakcji.

💡
Wyobraź sobie żarówkę
Żarówka może być włączona (1) albo wyłączona (0). Komputer ma miliardy takich "żarówek" — nazywają się tranzystorami. Każdy tranzystor to jeden bit — jeden fragment informacji. Twój procesor ma ich dziś ponad 50 miliardów — wszystkie na kawałku krzemu wielkości paznokcia.
0️⃣1️⃣
Bit — atom informacji
Najmniejsza możliwa porcja informacji. Może mieć wartość 0 (brak prądu) albo 1 (jest prąd). Nic pomiędzy. Dosłownie wszystko w komputerze — zdjęcia, muzyka, kod — to ciągi zer i jedynek.
📦
Bajt — 8 bitów razem
8 bitów daje bajt. 8 żarówek ułożonych w rząd to 256 różnych kombinacji (2⁸ = 256). Bajtem możemy zakodować jedną literę, jedną liczbę 0–255, jeden piksel w bardzo prostej grafice.
🔢
Jak wygląda litera "A"?
W systemie ASCII litera "A" to liczba 65. W binarnym: 01000001. Komputer nie wie co to "A" — widzi tylko ten wzór prądu. To my nadaliśmy mu znaczenie.
🤯 Kluczowa myśl
Wszystko co widzisz na ekranie to zera i jedynki. Zdjęcie Twojej rodziny? Ciąg bitów. Piosenka na Spotify? Ciąg bitów. Ten tekst który czytasz? Ciąg bitów. Cały internet? Bity lecące przez kabelki i powietrze. Programowanie to sztuka nadawania tym bitom znaczenia.
02 / 09 🧱 Warstwy abstrakcji

Abstrakcja — ukrywanie złożoności

Nikt nie programuje w zerach i jedynkach. Nikt nie myśli o tranzystorach pisząc aplikację. Przez dziesiątki lat informatycy budowali warstwy abstrakcji — każda warstwa ukrywa skomplikowane szczegóły niższej i daje prostsze narzędzia wyższej.

🚗
Analogia z samochodem
Kiedy skręcasz kierownicą w lewo — nie musisz wiedzieć jak działa układ hydrauliczny wspomagania, jak przekładnia kierownicza przenosi siłę, jak każde koło obraca się pod innym kątem (geometria Ackermanna). Po prostu skręcasz kółko i auto skręca. Kierownica to abstrakcja nad skomplikowaną mechaniką. W programowaniu działamy identycznie — abstrakcje na abstrakcjach.

💡 Dlaczego to ważne?

03 / 09 💬 Języki programowania

Czym jest język programowania?

Język programowania to umówiony sposób wydawania rozkazów komputerowi — zapisany tak, żeby ludzie mogli go czytać i pisać. Komputer nie rozumie angielskiego ani polskiego, ale rozumie precyzyjną składnię języka — i kompilator tłumaczy ją na bity.

👨‍🍳
Analogia z przepisem kulinarnym
Przepis to algorytm — lista kroków do wykonania w określonej kolejności. "Zagotuj wodę. Dodaj makaron. Gotuj 8 minut. Odcedź." Komputer to robot kuchenny który wykonuje przepis idealnie dosłownie — bez żadnej improwizacji, bez zdrowego rozsądku. Jeśli napiszesz "gotuj 8 minut" — ugotuje dokładnie 8 minut. Napisze warto gotować "do miękkości" — nie zrozumie.
🌐 Języki wysokiego poziomu
Python, JavaScript, Java, C# — blisko języka ludzkiego, daleko od sprzętu. Łatwe do nauki, mało kontroli nad szczegółami. Kompilator/interpreter robi za nas dużo pracy (zarządzanie pamięcią, typy itp.).
⚙️ Języki niskiego poziomu
C, C++, Rust, Asembler — blisko sprzętu, dużo kontroli. Musisz sam zarządzać pamięcią, typami danych. Trudniejsze, ale szybsze i precyzyjniejsze. Systemy operacyjne, sterowniki, gry AAA — pisane w C/C++.
Ten sam algorytm — 3 języki 
# Python — bardzo czytelny
def powitaj(imie):
    print("Cześć, " + imie)

powitaj("Ania")

// JavaScript — podobnie
function powitaj(imie) {
    console.log("Cześć, " + imie);
}

// Java — bardziej ceremonialny
public static void powitaj(String imie) {
    System.out.println("Cześć, " + imie);
}

// Asembler x86 — to samo, inaczej
mov  rdi, format_str
mov  rsi, name_ptr
call printf          ; wywołaj printf
🤔 Skąd wiedzieć jaki język wybrać?
Web frontend? JavaScript (musisz, bo przeglądarka nic innego nie rozumie). Data Science / AI? Python (ekosystem bibliotek). Android? Kotlin / Java. iOS? Swift. Backend enterprise? Java, C#. Wydajność / systemy? C, C++, Rust. Na start? Python — prosta składnia, natychmiastowe efekty, ogromna społeczność.
04 / 09 📦 Zmienne i typy danych

Zmienne — pudełka na dane

Zmienna to nazwane miejsce w pamięci komputera gdzie przechowujesz jakąś wartość. Wyobraź sobie pudełko z etykietą — możesz włożyć do niego coś, wyjąć, zamienić na inne. Komputer nie wie co to "imię" czy "wiek" — ale Ty możesz dać zmiennej nazwę która ma sens dla człowieka.

📬
Analogia ze skrzynkami pocztowymi
Wyobraź sobie rząd skrzynek pocztowych w bloku. Każda ma numer (adres w pamięci) i imię właściciela (nazwa zmiennej). Możesz włożyć list do skrzynki "Jan" — nie musisz znać numeru, wystarczy imię. Komputer tłumaczy imię na adres. Kiedy program się kończy — skrzynki są opróżniane (pamięć zwalniana).
Zmienne w Javie — przykłady 
// Deklaracja zmiennej: TYP nazwa = wartość;
int     wiek       = 25;          // liczba całkowita
double  wzrost     = 1.78;        // liczba z przecinkiem
String  imie       = "Ania";      // tekst (łańcuch znaków)
boolean czyPelnoletni = true;    // prawda lub fałsz
char    pierwszaLitera = 'A';    // jeden znak

// Zmiana wartości — nowe pudełko nie, nowa zawartość tak
wiek = 26;        // teraz wiek to 26
imie = "Kasia";   // teraz imie to "Kasia"

// Operacje na zmiennych
int a = 10;
int b = 3;
int suma     = a + b;  // 13
int roznica  = a - b;  // 7
int iloczyn  = a * b;  // 30
int iloraz   = a / b;  // 3 (całkowite!)
int reszta   = a % b;  // 1 (modulo = reszta z dzielenia)
🔢
Typy liczbowe
int — całkowite (−2mld do +2mld)
long — wielkie całkowite
double — ułamkowe (64 bity)
float — ułamkowe (32 bity)

Dlaczego różne? Bo zajmują różną ilość pamięci i mają różną precyzję.
📝
Tekst i znaki
char — jeden znak (litera, cyfra, emoji)
String — ciąg znaków (słowo, zdanie, email)

String to tak naprawdę tablica charów — "Ania" to ['A','n','i','a'] pod spodem. Dlatego można liczyć długość, wyciągać litery itp.
✅
Prawda / fałsz
boolean — tylko dwie wartości: true lub false

To podstawa każdej decyzji w programie. "Czy użytkownik jest zalogowany? Czy liczba jest większa od 10? Czy email zawiera @?" — wszystko to pytania na które odpowiedź to boolean.
🧠 Dlaczego typy danych mają znaczenie?
Komputer musi wiedzieć z góry ile miejsca zarezerwować w pamięci i jak interpretować bity. Liczba 65 jako int to 65. Te same bity jako char to litera "A". Te same bity jako float to jakaś dziwna liczba zmiennoprzecinkowa. Bity nie mają znaczenia — mają je dopiero gdy wiemy jak je interpretować.
05 / 09 🔀 Logika i warunki

If / Else — komputer podejmuje decyzje

Program który wykonuje tylko ciąg kroków od góry do dołu jest mało użyteczny. Prawdziwa moc zaczyna się gdy program może podejmować decyzje w zależności od danych. Do tego służą instrukcje warunkowe — if/else.

🚦
Analogia z sygnalizacją świetlną
Kierowca patrzy na sygnalizator: JEŚLI światło jest zielone — TO jedź. W PRZECIWNYM RAZIE — stój. To dokładnie instrukcja if/else. Komputer sprawdza warunek (jest to pytanie z odpowiedzią true/false) i wykonuje jeden z dwóch bloków kodu.
If / Else — przykłady 
int wiek = 17;

// Podstawowy if/else
if (wiek >= 18) {
    System.out.println("Wejdź na seans!");
} else {
    System.out.println("Przykro mi, musisz mieć 18+");
}
// wydrukuje: "Przykro mi, musisz mieć 18+"

// Wiele warunków — else if
int wynik = 75;

if (wynik >= 90) {
    System.out.println("Celujący!");
} else if (wynik >= 75) {
    System.out.println("Bardzo dobry");
} else if (wynik >= 50) {
    System.out.println("Dostateczny");
} else {
    System.out.println("Niedostateczny");
}
// wydrukuje: "Bardzo dobry"

// Łączenie warunków: && (i), || (lub), ! (nie)
boolean maKarte = true;
boolean maGotowke = false;

if (maKarte || maGotowke) {
    System.out.println("Może zapłacić");
}
// true LUB false = true → wydrukuje "Może zapłacić"
🔗
Operatory logiczne
&& (AND / I) — oba warunki muszą być prawdziwe
true && true = true
true && false = false

|| (OR / LUB) — przynajmniej jeden musi być prawdziwy
false || true = true
false || false = false

! (NOT / NIE) — odwraca wartość
!true = false
⚠️ Częsty błąd
W Javie (i wielu innych językach) = to przypisanie (wkładam wartość do zmiennej), a == to porównanie (czy są równe?). Napisanie if (wiek = 18) zamiast if (wiek == 18) to klasyczny błąd który prowadzi do dziwnych zachowań.
06 / 09 🔄 Pętle — powtarzanie

Pętle — rób to samo X razy

Komputer jest idealny do powtarzania tych samych czynności wiele razy — szybko i bez znudzenia. Pętle to mechanizm który mówi "wykonaj ten blok kodu dopóki warunek jest spełniony" albo "wykonaj go dla każdego elementu listy".

🧹
Analogia ze sprzątaniem
"Weź każde naczynia ze stołu i umyj je." — to pętla for-each. Bierzesz kolejne naczynia ze stosu (lista) i dla każdego wykonujesz tę samą czynność (mycie). Albo: "Odkurzaj dopóki worek nie będzie pełny." — to pętla while. Robisz coś dopóki warunek (worek niepełny) jest prawdziwy.
3 rodzaje pętli w Javie 
// ══ FOR — gdy wiesz ile razy ══════════════════════
// for (start; warunek_kontynuacji; krok_po_każdej_iteracji)
for (int i = 0; i < 5; i++) {
    System.out.println("Krok numer: " + i);
}
// wydrukuje: 0, 1, 2, 3, 4  (i++ zwiększa i o 1 po każdym kroku)

// ══ WHILE — gdy nie wiesz ile razy ═══════════════
int liczba = 1;
while (liczba <= 100) {
    liczba = liczba * 2;  // podwajaj
}
System.out.println(liczba); // 128 — pierwsza potęga 2 > 100

// ══ FOR-EACH — dla każdego elementu listy ════════
String[] owoce = {"jabłko", "gruszka", "banan"};

for (String owoc : owoce) {
    System.out.println("Mam: " + owoc);
}
// wydrukuje: "Mam: jabłko", "Mam: gruszka", "Mam: banan"

// ══ BREAK i CONTINUE ════════════════════════════
for (int i = 0; i < 10; i++) {
    if (i == 3) continue; // pomiń i=3, idź do następnego
    if (i == 7) break;    // zatrzymaj pętlę przy i=7
    System.out.println(i);   // wydrukuje: 0,1,2,4,5,6
}
🔥 Nieskończona pętla — klasyczna katastrofa
while(true) { /* coś */ } — pętla która nigdy się nie kończy. Procesor kręci się w kółko, program się zawiesza, wentylatory laptopa zaczynają wyć. Zawsze upewnij się że pętla ma warunek który kiedyś będzie fałszywy i że coś w pętli go zmienia!
07 / 09 🔧 Funkcje — kawałki kodu z nazwą

Funkcje — nie powtarzaj się

Wyobraź sobie że za każdym razem gdy chcesz zagotować wodę, musisz od nowa pisać wszystkie kroki: napełnij czajnik, podłącz do prądu, czekaj aż zagotuje... Funkcja to przepis z nazwą — piszesz go raz, a potem wywołujesz po nazwie ile razy chcesz.

🧰
Analogia z przyciskiem w windzie
Kiedy naciskasz "3" w windzie — nie musisz wiedzieć jak działają silniki, liny, bezpieczniki. Naciśnięcie przycisku = wywołanie funkcji. Winda wie co robić. Możesz nacisnąć "3" sto razy — za każdym razem ta sama sekwencja się wykona. Możesz też nacisnąć "5" — ta sama funkcja, inny argument.
Funkcje — anatomia i przykłady 
// Anatomia funkcji w Javie:
// zwracanyTyp nazwaFunkcji(parametry) { ciało }

// Funkcja bez parametrów, bez zwracania wartości
void przywitajSie() {
    System.out.println("Cześć, świecie!");
}

// Funkcja z parametrem
void przywitajOsobe(String imie) {
    System.out.println("Cześć, " + imie + "!");
}

// Funkcja która ZWRACA wartość (return)
int dodaj(int a, int b) {
    return a + b;  // zwróć wynik do wywołującego
}

// Wywołanie funkcji:
przywitajSie();               // "Cześć, świecie!"
przywitajOsobe("Ania");       // "Cześć, Ania!"
przywitajOsobe("Kacper");     // "Cześć, Kacper!"

int wynik = dodaj(10, 5);     // wynik = 15
int wynik2 = dodaj(100, 200); // wynik2 = 300

// Funkcja bardziej realistyczna
boolean czyPelnoletni(int wiek) {
    return wiek >= 18;
}

if (czyPelnoletni(16)) {
    System.out.println("Wejdź");
} else {
    System.out.println("Nie możesz wejść");
}
1
Deklaracja — piszesz przepis
Tworzysz funkcję z jej nazwą, parametrami i ciałem. Kod w środku jeszcze się nie wykonuje — to tylko przepis czekający na wywołanie.
2
Wywołanie — uruchamiasz przepis
Piszesz nazwaFunkcji(argumenty) w dowolnym miejscu kodu. Program "skacze" do ciała funkcji, wykonuje je i wraca.
3
Parametry — wejście do funkcji
Wartości które "podajesz" do funkcji przy wywołaniu. Jak składniki przepisu — dodaj(3, 7) — 3 i 7 to argumenty.
4
Return — wyjście z funkcji
Funkcja może zwrócić wynik. return x kończy działanie i przekazuje wartość x tam skąd funkcja była wywołana.
08 / 09 🏗️ Klasy i obiekty — OOP

Klasy — szablony dla rzeczy

Do tej pory trzymaliśmy dane w oddzielnych zmiennych — imię w jednym miejscu, wiek w innym. Gdy masz 1000 użytkowników, robi się bałagan. Klasy pozwalają grupować dane i funkcje które na nich działają w jeden spójny "typ" — własny, wymyślony przez programistę.

🏭
Analogia z fabryką samochodów
Klasa to projekt/schemat — rysunek techniczny samochodu. Mówi jak samochód wygląda (ma silnik, 4 koła, kolor) i co potrafi robić (jedź, hamuj, zatankuj). Projekt sam w sobie nie jedzie. Obiekt to konkretny egzemplarz — samochód który zjeżdża z taśmy na podstawie tego projektu. Możesz z jednego projektu wyprodukować milion samochodów — każdy z własnym kolorem i przebiegiem.
Klasa Pies — definicja i użycie 
// ══ DEFINICJA KLASY (schemat/projekt) ════════════
class Pies {
    // POLA — dane które każdy pies posiada
    String imie;
    String rasa;
    int    wiek;
    double wagaKg;

    // KONSTRUKTOR — jak "urodzić" nowego psa
    Pies(String imie, String rasa, int wiek) {
        this.imie  = imie;   // this = ten konkretny pies
        this.rasa  = rasa;
        this.wiek  = wiek;
    }

    // METODY — co pies potrafi robić
    void szczekaj() {
        System.out.println(this.imie + ": Hau! Hau!");
    }

    void przedstawSie() {
        System.out.println("Jestem " + imie +
          ", rasa: " + rasa +
          ", wiek: " + wiek);
    }

    boolean czyMlody() {
        return wiek < 3;
    }
}

// ══ TWORZENIE OBIEKTÓW (egzemplarzy) ════════════
Pies reks = new Pies("Reks", "Owczarek", 5);
Pies burek = new Pies("Burek", "Kundelek", 2);

// ══ UŻYWANIE OBIEKTÓW ════════════════════════════
reks.szczekaj();        // "Reks: Hau! Hau!"
burek.przedstawSie();  // "Jestem Burek, rasa: Kundelek..."

System.out.println(burek.czyMlody()); // true (wiek=2 < 3)
System.out.println(reks.czyMlody());  // false (wiek=5)
🧬
4 filary OOP
Enkapsulacja — chowaj dane w środku klasy, udostępniaj tylko to co trzeba (prywatne pola, publiczne metody)

Dziedziczenie — klasa "Pies" może dziedziczyć z "Zwierzę" — przejmuje wszystkie właściwości i może dodać własne

Polimorfizm — każde zwierzę ma metodę wydajDzwiek() ale pies szczeka, kot miauczy, krowa muczy

Abstrakcja — ukrywasz złożoność, pokazujesz tylko interface
💬 Klasa vs Obiekt
Klasa = przepis, projekt, schemat. Istnieje w kodzie, nie zajmuje pamięci na dane.
Obiekt = konkretny egzemplarz stworzony na podstawie klasy. Zajmuje pamięć, ma własne wartości pól.

Pies — klasa (jeden egzemplarz w kodzie)
reks, burek — obiekty (dwa egzemplarze w pamięci)
09 / 09 🌐 Od kodu do ekranu

Jak kod staje się działającą aplikacją?

Piszesz kod w edytorze — i pojawia się aplikacja na ekranie. Między tymi dwoma faktami dzieje się dużo. Oto mapa podróży Twojego kodu — od tekstu w pliku przez kompilację, aż do bitów lecących przez kabelek do ekranu.

1
Piszesz kod w edytorze (np. VS Code, IntelliJ)
Tworzysz plik tekstowy z rozszerzeniem .java, .py, .js itp. To zwykły tekst — komputer go jeszcze nie rozumie. Edytor koloruje składnię (syntax highlighting) ale to tylko dla Twoich oczu.
2
Kompilator / Interpreter tłumaczy kod
Kompilator (Java, C++) — czyta cały plik i tłumaczy na kod maszynowy lub bytecode jednorazowo. Tworzysz plik wykonywalny .exe albo .class. Błędy są wykrywane przed uruchomieniem.
Interpreter (Python, JavaScript) — czyta i wykonuje kod linia po linii na żywo. Błędy wychodzą dopiero podczas działania.
3
System Operacyjny ładuje program do pamięci RAM
OS rezerwuje kawałek RAM dla Twojego programu, ładuje do niego kod i dane, ustawia stos wywołań. Twój program dostaje swój "piaskownicę" — własny obszar pamięci, odizolowany od innych programów.
4
Procesor wykonuje instrukcje jedna po drugiej
CPU pobiera instrukcje z RAM, dekoduje je (co to za instrukcja?), wykonuje (ALU: dodaj, porównaj, skocz) i zapisuje wynik. Miliardy instrukcji na sekundę. Każda instrukcja to kilka nanosekund.
5
Karta graficzna renderuje piksele
Gdy program mówi "narysuj ten przycisk tutaj" — OS przekazuje to GPU. Karta graficzna przelicza miliony pikseli i wypycha je na monitor przez kabel DisplayPort/HDMI. To co widzisz to fotony odbijające się od ciekłokrystalicznych elementów sterowanych napięciem.
🌐 Jak działa strona WWW?
Wpisujesz google.com → DNS zamienia na adres IP → Twój komputer wysyła żądanie HTTP przez internet → Serwer Google dostaje żądanie, kompiluje HTML/CSS/JS → Przesyła je z powrotem jako bajty → Twoja przeglądarka parsuje HTML, buduje DOM, renderuje piksele → Widzisz stronę. Wszystko w <200 milisekund.
📱 Jak działa aplikacja mobilna?
Pobierasz .apk / .ipa ze sklepu → System instaluje, sprawdza uprawnienia → Klikasz ikonkę → OS tworzy nowy proces, ładuje kod do RAM → JVM (Android) / ART kompiluje bytecode do kodu natywnego → Wyświetla interfejs przez framework UI → CPU+GPU renderuje 60 klatek na sekundę.

🏆 Masz już fundament

Teraz rozumiesz że komputer to nic magicznego — to miliard przełączników które świecą lub nie świecą, ułożonych w sprytny sposób. Zrozumiałeś warstwy abstrakcji — każda warstwa ukrywa złożoność i daje prostsze narzędzia. Wiesz że zmienna to pudełko w pamięci, funkcja to nazwany przepis, a klasa to szablon dla danych i akcji. Wiesz jak kod przechodzi przez kompilator i ląduje jako piksele na ekranie. To jest fundament na którym buduje się całe programowanie — od prostych skryptów po systemy obsługujące miliony użytkowników.
