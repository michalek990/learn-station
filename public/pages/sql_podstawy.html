02
Klucze i relacje
jak tabele się ze sobą łączą
🔑 Rodzaje kluczy
PRIMARY KEY PK Unikalny identyfikator wiersza. Nie może być NULL. Jeden na tabelę.
FOREIGN KEY FK Wskazuje na PK innej tabeli. Zapewnia spójność danych (integralność referencyjna).
UNIQUE UQ Wartości muszą być unikalne, ale może być NULL. Może być wiele na tabelę.
NOT NULL NN Kolumna nie może mieć wartości NULL. Obowiązkowe pole.
INDEX Przyspiesza wyszukiwanie. Nie wpływa na dane, tylko na wydajność.
CHECK Walidacja wartości. np. wiek > 0 AND wiek < 150
CREATE TABLE
CREATE TABLE uzytkownicy (
  id        INT PRIMARY KEY AUTO_INCREMENT,
  email     VARCHAR(255) NOT NULL UNIQUE,
  imie      VARCHAR(100) NOT NULL,
  wiek      INT CHECK (wiek > 0),
  created   TIMESTAMP DEFAULT NOW()
);

CREATE TABLE zamowienia (
  id          INT PRIMARY KEY AUTO_INCREMENT,
  uzytkownik_id INT NOT NULL,
  kwota       DECIMAL(10,2) NOT NULL,
  data        DATE NOT NULL,

  -- FK łączy z tabelą uzytkownicy
  FOREIGN KEY (uzytkownik_id)
    REFERENCES uzytkownicy(id)
    ON DELETE CASCADE
);
🔗 Typy relacji
1 do 1 (1:1)
Uzytkownik ──── Profil Jeden użytkownik ma jeden profil. Rzadko używane, często łączy się w jedną tabelę.
1 do wielu (1:N)
Uzytkownik │ ├── Zamowienie 1 ├── Zamowienie 2 └── Zamowienie 3 Jeden user ma wiele zamówień. Najczęstsza relacja!
Wiele do wielu (N:M)
Student ── student_kurs ── Kurs Wymaga tabeli pośredniej (junction)! Student na wielu kursach, kurs ma wielu studentów.
03
SELECT — pobieranie danych
podstawowe i zaawansowane zapytania
SELECT podstawy
-- Wszystkie kolumny (unikaj w produkcji!)
SELECT * FROM uzytkownicy;

-- Konkretne kolumny
SELECT imie, email, wiek
FROM uzytkownicy;

-- Aliasy kolumn (AS)
SELECT
  imie  AS name,
  email AS mail,
  wiek  AS age
FROM uzytkownicy;

-- Kolumna obliczana
SELECT
  imie,
  cena * 1.23 AS cena_z_vat
FROM produkty;
WHERE — filtrowanie
-- Proste warunki
SELECT * FROM uzytkownicy
WHERE wiek > 18;

-- Wiele warunków
SELECT * FROM uzytkownicy
WHERE wiek > 18 AND wiek < 65;

-- IN — jedna z wartości
SELECT * FROM produkty
WHERE kategoria IN ('buty', 'kurtki');

-- BETWEEN — zakres
SELECT * FROM zamowienia
WHERE kwota BETWEEN 100 AND 500;

-- LIKE — wzorzec tekstowy
SELECT * FROM uzytkownicy
WHERE imie LIKE 'Jan%';
-- % = dowolna liczba znaków
-- _ = dokładnie jeden znak

-- IS NULL
SELECT * FROM uzytkownicy
WHERE telefon IS NULL;
Kolejność klauzul SELECT: SELECT → FROM → WHERE → GROUP BY → HAVING → ORDER BY → LIMIT. Musisz je pisać dokładnie w tej kolejności!
04
JOIN — łączenie tabel
pobieranie danych z wielu tabel naraz
INNER JOIN
Tylko wiersze które mają dopasowanie w obu tabelach. Najpopularniejszy.
LEFT JOIN
Wszystko z lewej tabeli + dopasowania z prawej. Brak dopasowania = NULL.
RIGHT JOIN
Wszystko z prawej tabeli + dopasowania z lewej. Rzadko używany.
FULL OUTER JOIN
Wszystko z obu tabel. Brak dopasowania = NULL po obu stronach.
przykłady JOIN
-- INNER JOIN: tylko dopasowane
SELECT
  u.imie,
  z.kwota,
  z.data
FROM uzytkownicy u
INNER JOIN zamowienia z
  ON u.id = z.uzytkownik_id;

-- LEFT JOIN: wszyscy userzy,
-- nawet bez zamówień
SELECT
  u.imie,
  COUNT(z.id) AS ile_zamowien
FROM uzytkownicy u
LEFT JOIN zamowienia z
  ON u.id = z.uzytkownik_id
GROUP BY u.id;

-- JOIN 3 tabel
SELECT
  u.imie,
  z.kwota,
  p.nazwa AS produkt
FROM uzytkownicy u
JOIN zamowienia z
  ON u.id = z.uzytkownik_id
JOIN produkty p
  ON z.produkt_id = p.id;
05
INSERT / UPDATE / DELETE
modyfikacja danych — DML (Data Manipulation Language)
INSERT
-- Jeden wiersz
INSERT INTO uzytkownicy
  (imie, email, wiek)
VALUES
  ('Jan', '[email protected]', 30);

-- Wiele wierszy naraz (wydajniej!)
INSERT INTO uzytkownicy
  (imie, email, wiek)
VALUES
  ('Anna', '[email protected]', 25),
  ('Kuba', '[email protected]', 28),
  ('Ola',  '[email protected]',  22);

-- INSERT z SELECT (kopiowanie danych)
INSERT INTO archiwum (imie, email)
SELECT imie, email
FROM uzytkownicy
WHERE wiek > 60;
UPDATE / DELETE
-- UPDATE — zawsze z WHERE!
UPDATE uzytkownicy
SET
  imie  = 'Janek',
  email = '[email protected]'
WHERE id = 5;

-- UPDATE z obliczeniem
UPDATE produkty
SET cena = cena * 1.1
WHERE kategoria = 'elektronika';

-- DELETE — zawsze z WHERE!
DELETE FROM uzytkownicy
WHERE id = 5;

-- Usunięcie wszystkich wierszy
DELETE FROM sesje;
-- lub szybciej:
TRUNCATE TABLE sesje;
🚨 NIGDY nie rób UPDATE ani DELETE bez WHERE! UPDATE uzytkownicy SET haslo = 'abc' zmieni hasło WSZYSTKIM użytkownikom. Przed każdym UPDATE/DELETE najpierw napisz odpowiadający SELECT żeby zobaczyć co zmienisz.
06
GROUP BY i funkcje agregujące
grupowanie i obliczenia na zbiorach
∑ Funkcje agregujące
COUNT(*)Liczba wierszy w grupie
COUNT(kol)Liczba wierszy gdzie kolumna NOT NULL
SUM(kol)Suma wartości
AVG(kol)Średnia wartość
MIN(kol)Minimalna wartość
MAX(kol)Maksymalna wartość
GROUP_CONCATSklejenie wartości w string (MySQL)
GROUP BY przykłady
-- Ile zamówień ma każdy user?
SELECT
  uzytkownik_id,
  COUNT(*)     AS ile_zamowien,
  SUM(kwota)   AS suma,
  AVG(kwota)   AS srednia,
  MAX(kwota)   AS najwieksze
FROM zamowienia
GROUP BY uzytkownik_id;

-- Sprzedaż per kategoria per miesiąc
SELECT
  kategoria,
  MONTH(data)  AS miesiac,
  SUM(kwota)   AS sprzedaz
FROM zamowienia
GROUP BY kategoria, MONTH(data)
ORDER BY miesiac, sprzedaz DESC;
07
ORDER BY / HAVING / LIMIT
sortowanie, filtrowanie grup i ograniczanie wyników
ORDER BY i LIMIT
-- Sortowanie rosnąco (domyślne)
SELECT * FROM produkty
ORDER BY cena ASC;

-- Sortowanie malejąco
SELECT * FROM produkty
ORDER BY cena DESC;

-- Po wielu kolumnach
SELECT * FROM uzytkownicy
ORDER BY nazwisko ASC, imie ASC;

-- LIMIT — tylko N wierszy
SELECT * FROM produkty
ORDER BY sprzedaz DESC
LIMIT 10;  -- top 10!

-- LIMIT + OFFSET (paginacja)
SELECT * FROM produkty
ORDER BY id
LIMIT 10 OFFSET 20;
-- strona 3 (po 10 na stronę)
HAVING — filtrowanie po agregacji
-- WHERE filtruje WIERSZE
-- HAVING filtruje GRUPY (po GROUP BY)

-- Userzy z więcej niż 5 zamówieniami
SELECT
  uzytkownik_id,
  COUNT(*) AS ile
FROM zamowienia
GROUP BY uzytkownik_id
HAVING COUNT(*) > 5;

-- WHERE + HAVING razem
SELECT
  uzytkownik_id,
  SUM(kwota) AS suma
FROM zamowienia
WHERE data > '2024-01-01'
-- WHERE: tylko zamówienia z 2024
GROUP BY uzytkownik_id
HAVING SUM(kwota) > 1000
-- HAVING: tylko sumy > 1000
ORDER BY suma DESC;
WHERE vs HAVING: WHERE działa na surowych wierszach PRZED grupowaniem. HAVING działa na grupach PO grupowaniu. Nie możesz użyć COUNT(*) > 5 w WHERE — to musi być w HAVING!
